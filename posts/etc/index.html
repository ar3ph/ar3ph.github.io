Posted on May 26, 2025

Linux Everyday Software

Linux has its own set of everyday software, which is largely different than that of Windows and MacOS. My frequently used Linux software often has a developer nature and sometimes a server software taste. Here I’m listing a few of those and how I configure them uniquely.

VirtualBox

VirtualBox is a hypervisor on Linux that supports Linux, Windows and some other operating systems. I find it’s kind of awkward to pass through the webcam, so I made a simple script to remind myself how to do this:

# Oracle VM VirtualBox Extension Pack needed
VBoxManage list webcams
VBoxManage controlvm "win1" webcam attach .1

PostgreSQL

Sometimes I need a Postgres instance for testing my projects or I’d like to configure my database quickly. Here is a docker compose file for running PostgreSQL at 127.0.0.1:5432:

# docker-compose.yaml
version: '3.1'

services:
  db:
    image: postgres
    restart: unless-stopped
    environment:
      POSTGRES_PASSWORD: postgres
      PGDATA: /var/lib/postgresql/data/pgdata
    volumes:
      - './data/:/var/lib/postgresql/data/'
    ports:
      - 127.0.0.1:5432:5432

With this compose file, if you want to remove everything in the DB, just remove the data folder and restart the container, like so:

docker compose down
rm -rf data
docker compose up -d

This compose file also only listens at 127.0.0.1, so it’s not exposing the database to the inet(internet).

pi-hole

I have a docker compose file for pihole over dnscrypt-proxy. This was to configure an encrypted local DNS server, as the DNS server of my PC. However, pi-hole won’t start stably with this configuration, so I stopped using this. But here is the docker compose file anyway:

# docker-compose.yaml
version: "3"
services:
  dnscrypt:
    container_name: dnscrypt-proxy
    image: klutchell/dnscrypt-proxy:latest
    networks:
      pihole_net:
        ipv4_address: 172.16.1.2
    expose:
      - "5053/udp"
      - "5053/tcp"
    environment:
      TZ: 'Europe/London'
    dns:
      - 1.1.1.1
    restart: unless-stopped

  pihole:
    container_name: pihole
    image: pihole/pihole:latest
    networks:
      pihole_net:
        ipv4_address: 172.16.1.3
    ports:
      - "53:53/tcp"
      - "53:53/udp"
      - "80:80/tcp"
      - "443:443/tcp"
    environment:
      TZ: 'Europe/London'
      WEBPASSWORD: '123'
      DNS1: "172.16.1.2#5053"
      DNS2: "no"
      VIRTUAL_HOST: "172.16.1.3"
    volumes:
       - './etc-pihole/:/etc/pihole/'
       - './etc-dnsmasq.d/:/etc/dnsmasq.d/'
    dns:
      - 1.1.1.1
    restart: unless-stopped
    depends_on:
      - dnscrypt

networks:
  pihole_net:
    driver: bridge
    ipam:
      config:
        - subnet: 172.16.1.0/24

Certbot

Certbot is an SSL certificate tool. You can learn how to use it on the Internet. What’s interesting is that Certbot is now shipped with a systemd service to renew the certs automatically.

I edited the service with the following command to restart nginx if the certs are renewed:

sudo EDITOR=nvim systemctl edit certbot

The edited service looks like this:

### Editing /etc/systemd/system/certbot.service.d/override.conf
### Anything between here and the comment below will become the new contents of the file

ExecStart=/usr/bin/certbot -q renew --no-random-sleep-on-renew --renew-hook "systemctl reload nginx"

### Lines below this comment will be discarded

### /lib/systemd/system/certbot.service
# [Unit]
# Description=Certbot
# Documentation=file:///usr/share/doc/python-certbot-doc/html/index.html
# Documentation=https://certbot.eff.org/docs
# [Service]
# Type=oneshot
# ExecStart=/usr/bin/certbot -q renew --no-random-sleep-on-renew
# PrivateTmp=true

unbound

This is my local DNS server for now. The following configuration will start a DNS service at 127.0.0.1:53, with a little ad blocking.

# /etc/unbound/unbound.conf
# Unbound configuration file for Debian.
#
# See the unbound.conf(5) man page.
#
# See /usr/share/doc/unbound/examples/unbound.conf for a commented
# reference config file.
#
# The following line includes additional configuration files from the
# /etc/unbound/unbound.conf.d directory.
# include-toplevel: "/etc/unbound/unbound.conf.d/*.conf"
server:
port: 53
tls-upstream: no
tls-cert-bundle: "/etc/ssl/certs/ca-certificates.crt"
prefetch: yes
prefetch-key: yes
rrset-roundrobin: yes
do-not-query-localhost: yes
qname-minimisation-strict: yes
hide-identity: yes
hide-version: yes
cache-min-ttl: 300
cache-max-ttl: 3600
num-threads: 1
msg-cache-slabs: 2
rrset-cache-slabs: 2
infra-cache-slabs: 2
key-cache-slabs: 2
rrset-cache-size: 256m
msg-cache-size: 128m
so-rcvbuf: 8m

local-zone: "doubleclick.net" redirect
local-data: "doubleclick.net A 127.0.0.1"
local-zone: "googlesyndication.com" redirect
local-data: "googlesyndication.com A 127.0.0.1"
local-zone: "googleadservices.com" redirect
local-data: "googleadservices.com A 127.0.0.1"
local-zone: "google-analytics.com" redirect
local-data: "google-analytics.com A 127.0.0.1"
local-zone: "ads.youtube.com" redirect
local-data: "ads.youtube.com A 127.0.0.1"
local-zone: "adserver.yahoo.com" redirect
local-data: "adserver.yahoo.com A 127.0.0.1"

forward-zone:
name: "."
forward-addr:  1.1.1.1@53
