Repository

What are dotfiles

Dotfiles are conventional config files for UNIX like operating systems. These filenames and directory names start with a leading dot (.), hence the name, dotfiles. In my case, the dotfiles I’m sharing are specifically for Linux, but MacOS also conforms to this convention.

Dotfiles Usage

Nowadays, I’m a KDE user and don’t use the old sway config anymore, so in the repository, what are interesting to you are zshrc, zprofile, gitconfig and config/home-manager.

I took out the leading dot from the files in my git repo, so zshrc should be called .zshrc in the home directory. If you want to try out my dotfiles, copy them to your home directory and rename them accordingly.

Configured Programs

Here is a detailed view to every configured program.

Zsh

Zsh is configured by zshrc and zprofile. Let’s look at zprofile first.

zprofile

Firstly, the default web browser is configured:

for b in firefox librewolf chromium-browser chromium brave;do
        hash "$b" >/dev/null 2>&1 && {
                export BROWSER="$b"
                break
        }
done

This cycles through firefox, librewolf, chromium and brave and if one of these browsers exist, this piece of code will set that as the default web browser.

Then using the same technique, the default text editor is configured:

for e in nvim vim nano vi emacs; do
        hash "$e" >/dev/null 2>&1 && {
                export EDITOR="$e"
                break
        }
done

And next is the changes to PATHs:

export PATH="$HOME/.local/bin:$PATH"
export GOPATH="$HOME/go"
export PATH="${GOPATH}/bin:${PATH}"
[ -d "$HOME/.ghcup" ] && export PATH="$HOME/.ghcup/bin:$PATH"

Here programs in ~/.local/bin directory and go and haskell(ghc) programs are included in the execution path.

Then the language and encoding are configured. Although these environment variables are usually automatically set, there are occasions where these manual settings are necessary.

export LANG="en_US.UTF-8"
export LC_COLLATE="C"
export LC_ALL="en_US.UTF-8"

The next line enables neovim’s true color support:

export NVIM_TUI_ENABLE_TRUE_COLOR=1

This line is useful for i3wm and bspwm, mainly fixing the window focusing issue for the JetBrains suite:

export _JAVA_AWT_WM_NONREPARENTING=1

If the display system is wayland, notify Mozilla and Qt applications to use wayland:

[ -z "$WAYLAND_DISPLAY" ] && [ "$XDG_VTNR" -eq 1 ] >/dev/null 2>&1 && {
        export MOZ_ENABLE_WAYLAND=1
        export XDG_SESSION_TYPE=wayland
        export QT_QPA_PLATFORM=wayland
}

At last, load the local zprofile (located at ~/.zprofile.local) if it exists:

[ -f "$HOME/.zprofile.local" ] && source "$HOME/.zprofile.local"

zshrc

Firstly the command history is configured:

SAVEHIST=1000000
HISTFILE=~/.zsh_history
touch $HISTFILE

Next there are PATH and default editor settings, which will conflict with the zprofile a little bit:

export PATH="$HOME/.bin:$PATH"
export PATH="$HOME/.config/emacs/bin:$PATH"
export EDITOR=nvim
export PATH="$HOME/.local/bin:$PATH"
export PATH="${GOPATH}/bin:${PATH}"
export GOPATH="$HOME/go"

For a better development environment, I loaded:

  • GHC (haskell environment)
  • asdf-vm (development environment version manager)
  • pnpm and yarn (alternative npm)
  • direnv (autoloads a script when entering a directory)
  • poetry (python dependency tool)
# ------ devenv start
# zfunc
ZFUNC_DIR="$HOME/.config/zsh/zfunc"
[ -d ${ZFUNC_DIR} ] || mkdir -p ${ZFUNC_DIR}
fpath+=${ZFUNC_DIR}
# zfunc
# ---- ghc
[ -d "$HOME/.ghcup" ] && export PATH="$HOME/.ghcup/bin:$PATH"
# ---- ghc
# ---- ASDF VM
test -x "$(which asdf 2>/dev/null)" >/dev/null 2>&1 && {
        export PATH="${ASDF_DATA_DIR:-$HOME/.asdf}/shims:$PATH"
        mkdir -p "${ASDF_DATA_DIR:-$HOME/.asdf}/completions"
        [ -f "${ASDF_DATA_DIR:-$HOME/.asdf}/completions/_asdf" ] || asdf completion zsh > "${ASDF_DATA_DIR:-$HOME/.asdf}/completions/_asdf"
        fpath=(${ASDF_DATA_DIR:-$HOME/.asdf}/completions $fpath)
}
# ---- ASDF VM
# ---- pnpm
hash pnpm >/dev/null 2>&1 && {
        export PNPM_HOME="$HOME/.local/share/pnpm"
        case ":$PATH:" in
                *":$PNPM_HOME:"*) ;;
                *) export PATH="$PNPM_HOME:$PATH" ;;
        esac
}
# ---- pnpm

# ---- yarn
hash yarn >/dev/null 2>&1 && {
        export PATH="$HOME/.yarn/bin:$PATH"
}
# ---- yarn

# ---- direnv
hash direnv >/dev/null 2>&1 && {
        eval "$(direnv hook zsh)"
}
# ---- direnv

# ---- python start
# ----- poetry start
command -v poetry >/dev/null 2>&1 && [ ! -f "$HOME/.config/zsh/zfunc/_poetry" ] && {
    mkdir -p ~/.config/zsh
        poetry completions zsh > $HOME/.config/zsh/zfunc/_poetry
}
# ----- poetry end
# ---- python end

# ------ devenv end

Next is to initialize autocomplete and colors:

autoload -U colors && colors
autoload -U compinit && compinit

Then it configures extremely simplified vim simulation key bindings:

bindkey -v '^?' backward-delete-char
export KEYTIMEOUT=1
bindkey -v
function zle-keymap-select {
  if [[ ${KEYMAP} == vicmd ]] ||
     [[ $1 = 'block' ]]; then
    echo -ne '\e[1 q'
  elif [[ ${KEYMAP} == main ]] ||
       [[ ${KEYMAP} == viins ]] ||
       [[ ${KEYMAP} = '' ]] ||
       [[ $1 = 'beam' ]]; then
    echo -ne '\e[5 q'
  fi
}
zle -N zle-keymap-select
zle-line-init() {
    zle -K viins
    echo -ne "\e[5 q"
}
zle -N zle-line-init
echo -ne '\e[5 q'
preexec() { echo -ne '\e[5 q' ;}
bindkey '^[[P' delete-char
zstyle ':completion:*' menu select
zstyle ':completion:*' rehash true
zmodload zsh/complist
bindkey -M menuselect 'h' vi-backward-char
bindkey -M menuselect 'k' vi-up-line-or-history
bindkey -M menuselect 'l' vi-forward-char
bindkey -M menuselect 'j' vi-down-line-or-history

Use ctrl+x ctrl+e to edit the current command:

autoload edit-command-line; zle -N edit-command-line
bindkey  ^x^e edit-command-line

Set the prompt, and if starship is available, use the prompt from starship:

autoload -U colors && colors
PROMPT="%B%{$fg[green]%}[%{$fg[green]%}%n%{$fg[green]%}@%{$fg[magenta]%}%M %{$fg[green]%}%~%{$fg[green]%}]%{$reset_color%}%b [RET=%?]"
PROMPT+="
$ "
export PROMPT

hash starship >/dev/null 2>&1 && eval "$(starship init zsh)"

The fast syntax highlight plugin, that is configured via home-manager:

syntaxhighlight=$HOME/.local/share/zsh/zsh-fast-syntax-highlighting/fast-syntax-highlighting.plugin.zsh
[ -f ${syntaxhighlight} ] && source ${syntaxhighlight}

This line fixes some weird key binding issues:

stty -ixon

If there is a local zshrc (located at ~/.zshrc.local), load it:

[ -f "$HOME/.zshrc.local" ] && source $HOME/.zshrc.local

git

gitconfig

Force all SSH connections use the key at ~/.ssh/id_github.

[core]
    sshCommand = ssh -i ~/.ssh/id_github -F /dev/null

Configure user info and default branch:

[user]
        email = mail@itsbobkat.local
        name = itsbobkat
[init]
        defaultBranch = main

Include custom git configs for certain directories:

[includeIf "gitdir:~/Git/College/"]
    path = ~/Git/College/gitconfig

[includeIf "gitdir:~/Git/GitHub/"]
    path = ~/Git/GitHub/gitconfig

[includeIf "gitdir:~/Git/Work/"]
    path = ~/Git/Work/gitconfig

Here are some aliases:

  • git utccommit: will make a commit with the current UTC time and sign it
  • git utccommitnosign: will make a commit with the current UTC time and do not sign it
  • git lg: shows a commit history graph
[alias]
    utccommit = !GPG_TTY=$(tty) git commit --date=\"$(date --utc +%Y-%m-%dT%H:%M:%S%z)\" -S
    utccommitnosign = !git commit --date=\"$(date --utc +%Y-%m-%dT%H:%M:%S%z)\"
    lg = "!f() { \
        git log --all --color --graph --pretty=format:'%C(bold yellow)<sig>%G?</sig>%C(reset) %C(red)%h%C(reset) -%C(yellow)%d%C(reset) %s %C(green)(%cr) %C(blue)<%an>%C(reset)' | \
        sed \
        -e 's#<sig>G</sig>#Good#' \
        -e 's#<sig>B</sig>#\\nBAD \\nBAD \\nBAD \\nBAD \\nBAD#' \
        -e 's#<sig>U</sig>#Unknown#' \
        -e 's#<sig>X</sig>#Expired#' \
        -e 's#<sig>Y</sig>#Expired Key#' \
        -e 's#<sig>R</sig>#Revoked#' \
        -e 's#<sig>E</sig>#Missing Key#' \
                -e 's#<sig>N</sig>#None#' |\
                more; \
    }; f"

Home Manager

This is a home directory manager based on the nix package system. Most of the config at config/home-manager is auto-generated. But I will point out the home.file directive to your attention. This copies Zsh’s syntax highlight plugin for nix store to the home directory:

  home.file = {
    ".local/share/zsh/zsh-fast-syntax-highlighting".source =
    "${pkgs.zsh-fast-syntax-highlighting}/share/zsh/site-functions";
  };
